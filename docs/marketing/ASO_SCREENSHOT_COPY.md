# ASO Screenshot Copy

Last updated: 2026-05-07

## Goal

Make the first six screenshots communicate one thing fast:

MicroBreaks is not a generic wellness app.
It is a desk-recovery product for eye strain, neck tension, posture fatigue, and focus drops.

## Copy rules

- Keep overlay copy short
- Lead with the symptom or immediate relief
- Avoid generic “productivity” language
- Avoid `AI`, `wellness`, `habit`, `gamification` in the first screenshots
- Show one job per screenshot

## Visual guidance

- Keep the top 35-40 percent of each screenshot clear enough for headline readability.
- Use one strong accent per frame, not rainbow callouts.
- Do not use too many arrows, circles, or annotation stickers.
- If the UI is already busy, reduce overlay copy further.

## Screenshot-specific notes

### 01-home-screen
- Prefer the state-based home surface.
- Show the current need selector and recommended reset card.

### 03-break-session
- Use a frame where the session is clearly in progress.
- The user should understand “guided” instantly.

### 04-exercises-library
- Prefer the outcome-packaged Breaks experience over a raw category list.

### 05-stats-dashboard
- Prefer the Recovery Story card and deeper signals over dense charts alone.

## Do not say

- `Boost productivity`
- `AI wellness coach`
- `Healthy habits`
- `Never forget a break again`
- `Gamified self-care`

## Prefer saying

- `Eye strain`
- `Desk neck`
- `Posture recovery`
- `Focus reset`
- `Weekly recovery story`
- `Work rhythm`
