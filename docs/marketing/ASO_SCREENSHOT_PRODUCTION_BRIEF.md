## Required exports

### iOS
- 6.7 inch: `1290 x 2796`
- 6.5 inch: `1284 x 2778`
- 5.5 inch: `1242 x 2208`

### Android
- phone portrait master between `1080 x 1920` and `1440 x 2560`

### Locales
- English
- Turkish

## Art direction

Use:
- one strong headline per frame
- one short subheadline
- calm top area for copy
- clear crop around the primary app surface

Avoid:
- too many badges
- too many arrows
- generic “wellness” words
- excessive decorative gradients
- fake 3D phone mockups that bury the real UI

## Typography

Recommended overlay stack:
- Headline: `Space Grotesk Bold`
- Subheadline: `Inter Medium`

## Color direction

Use the current marketing palette:
- primary `#0F766E`
- warm accent `#F59E0B`
- deep neutral `#10211D`
- light surface `#F8FBFA`

## Master composition rule

Top 35-40 percent of each frame should remain readable for copy.

The UI should do the narrative work.
The overlay copy should only sharpen the meaning.

## Screenshot-by-screenshot brief

### 01 Home

Source surface:
- state-based home screen

Must show:
- current need selector
- recommended reset card

Headline:
- `Eyes tired? Start here.`

Subheadline:
- `Pick the relief you need and begin a 2-minute reset.`

Visual note:
- use the cleanest state chip row
- avoid clutter from too many lower sections

### 02 Timer

Source surface:
- timer widget / work-break rhythm surface

Must show:
- work/break preset concept clearly

Headline:
- `Build a better work rhythm.`

Subheadline:
- `Use short work and recovery cycles that fit real screen-heavy days.`

Visual note:
- the timer should feel active, not idle

### 03 Guided session

Source surface:
- break session in progress

Must show:
- guided instruction
- step context

Headline:
- `Reset neck, posture, and breathing fast.`

Subheadline:
- `Guided desk-recovery sessions that fit between meetings and focus blocks.`

Visual note:
- the user should instantly understand this is not just a reminder app

### 04 Library

Source surface:
- Breaks screen with outcome-packaged framing

Must show:
- reset packs or outcome-first library view

Headline:
- `Go beyond starter relief.`

Subheadline:
- `Eye Rescue, Desk Neck Reset, Posture Rescue, Focus Reset, and more.`

Visual note:
- prefer the new outcome-pack framing over a raw category list

### 05 Recovery

Source surface:
- Stats screen with Recovery Story and deeper signals

Must show:
- Recovery Story card
- some signal or trend context

Headline:
- `See your weekly recovery story.`

Subheadline:
- `Know when strain is still building and what to improve next.`

Visual note:
- do not lead with dense charts alone

### 06 Trust

Source surface:
- Profile or settings surface with premium / support / privacy context

Must show:
- trust and continuity, not gamification clutter

Headline:
- `Stay private. Keep improving.`

Subheadline:
- `Private-by-default core with optional sync and a steadier reset rhythm.`

Visual note:
- keep this frame calm and trust-led

## QA checklist

- Copy does not overlap dynamic UI
- Headline readable at search-preview size
- Screenshot 1 works without reading screenshot 2
- Symptom language appears in the first 3 screenshots
- Recovery Story appears before any generic profile/gamification frame
- English and Turkish versions preserve hierarchy

## Rule

If a screenshot could belong to any generic wellness app, it is wrong.
